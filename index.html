Por qué nace este proyecto

Crear una web sencilla debería ser más fácil

Muchas personas quieren crear una página web básica, pero se pierden entre carpetas, archivos, hosting, GitHub y despliegue. Este proyecto intenta reducir esa barrera con una base simple, explicada paso a paso y pensada para aprender haciendo.

?

Problema

Demasiadas piezas al empezar

HTML, CSS, imágenes, carpetas, Git y publicación suelen aparecer a la vez, y eso genera bloqueo en quienes empiezan.

✓

Solución

Una plantilla base clara

El repositorio ofrece una web inicial, textos editables y una estructura sencilla para practicar sin depender de herramientas complejas.

↺

Enfoque

Aprender modificando

La idea no es vender un curso, sino enseñar cómo cambiar una web, guardar versiones y compartir mejoras de forma colaborativa.

💡

Qué aprenderás

Conceptos básicos con uso práctico real

1

Base web

Estructura de una página

Entenderás cómo se organizan cabecera, secciones, enlaces y contenido dentro de un documento HTML.

2

Contenido

Modificar textos e imágenes

Aprenderás a cambiar títulos, descripciones, llamadas a la acción, imágenes y elementos visibles sin romper la página.

3

Estilos

Ajustar colores y diseño

Verás cómo retocar estilos CSS para adaptar el aspecto visual a tu proyecto sin rehacer toda la maquetación.

4

Git y GitHub

Guardar versiones con seguridad

Practicarás lo básico de Git y GitHub para subir cambios, compararlos y mantener un historial claro del proyecto.

5

Publicación

Publicar una web sencilla

El objetivo final es que puedas tener una página visible y compartirla sin miedo a perder el control de tu trabajo.

Flujo de trabajo

Cinco pasos para llevar la web a producción

La meta es que entiendas el proceso completo: obtener la plantilla, cambiarla, guardar versiones, subirlas a GitHub y publicar una página funcional.

1

Descargar o clonar el proyecto

Empieza con una copia local del repositorio para poder editarla.

2

Modificar la plantilla

Cambia textos, imágenes, estilos y secciones según tu objetivo.

3

Guardar cambios con Git

Haz pequeños commits para mantener un historial comprensible.

4

Subir cambios a GitHub

Comparte el progreso y deja el proyecto accesible para revisarlo.

5

Publicar la web

Muestra el resultado final y sigue mejorándolo versión a versión.
